# Pre-Submission Checklist — High Cost AT Reports (NDIS)

This checklist is distilled from the multi-round QA review that was
carried out on the Bioness L300 Go report before submission. It is
generalised for reuse on **any** High Cost AT report, not just that
case. Run through it before any report is lodged.

## Evidence & citations

- [ ] **Every quantitative or research-derived claim in the body text has an inline citation with author and year.** (E.g. don't state a research-backed effect — "improved walking speed," "reduced falls risk" — without attributing it to a specific source at the point it's made.)
- [ ] **Any cited Minimum Detectable Change (MDC), Minimal Clinically Important Difference (MCID), or other normative/threshold value has a traceable, verifiable published source.** If no source can be confirmed, don't cite a number — reword as a plain descriptive finding (e.g. "a substantial participant-reported improvement") instead of implying a validated statistical threshold was exceeded.
- [ ] **The reference list is bibliographically complete and matches what's actually cited in the text** — no missing publication year, volume, issue, or page range; and no reference left in the list that isn't cited anywhere in the body (remove orphaned references rather than leaving them in "just in case").
- [ ] **"Clinically meaningful" or similar interpretive language is only used where a validated threshold for the relevant population has actually been cited.** Where it hasn't, use more conservative, defensible wording (e.g. "a substantial participant-specific improvement") instead.

## Legal / statutory wording

- [ ] **Any statutory or legal wording (e.g. references to a specific NDIS Act section) has been checked against the version of the Act currently in force, and against which funding framework actually applies to this participant's plan** (old framework vs new framework provisions can differ — don't assume one applies without checking). If unsure which framework applies, use general "applicable NDIS reasonable and necessary funding criteria" language rather than citing a specific section number.

## Device- and cost-specific justification

- [ ] **For any multi-component device, each component is individually justified with participant-specific evidence**, not just the device as a whole. Pre-empt the assessor's likely first question — "would the simpler/cheaper component alone have achieved this?" — by explicitly addressing what each component contributes and citing the specific evidence (trial data, washout data, clinical observation) that supports that component separately.
- [ ] **Value-for-money / cost-benefit claims are backed by concrete financial figures**, not clinical reasoning alone. Include total quoted cost, what's included in the package (fitting, programming, follow-up reviews, warranty, consumables/replacement parts), and anticipated ongoing costs — don't leave this to "refer to quotation" alone if the body text is making a value-for-money argument.
- [ ] **If the device itself captures usage analytics** (session logs, duration, step/cadence data, mode-switching data, etc.), **that data is included in the appendix as objective device-recorded evidence**, not left out in favour of participant self-report alone.

## Internal consistency

- [ ] **Every outcome-measure timepoint has a precise, explicitly stated date, and that date is consistent everywhere it's referenced** — across the outcome-measure section, any goal-alignment or summary section, and the corresponding appendix attachment. Don't leave ambiguous relative timing ("four weeks following the trial") when the underlying document (e.g. a washout period) has a different stated duration — reconcile or state the actual date.
- [ ] **Every numeric result (score, measurement, percentage change) is checked for consistency everywhere it appears in the document** — summary tables, in-text narrative, and any later summary/goal-alignment/recommendation sections must all state the exact same figure. A single mistyped digit repeated inconsistently is a common and easily-missed error.

## Administrative completeness

- [ ] **All placeholder fields are replaced with real values before the document is treated as submission-ready** — participant identifying details, dates, NDIS number — and a clinician declaration/signature block (name, professional title, qualifications, organisation, contact details, signature, date) is present. A report with placeholders still visible is not a finished submission regardless of how complete the clinical content is.
- [ ] **A final proofreading pass has been done for stray formatting artifacts** — leftover stray characters (e.g. trailing backslashes from markdown/copy-paste), words or lines split apart by page-break editing, duplicated sentences from a copy/paste edit, and inconsistent use of the participant's name (no leftover initials or wrong name once a report has been de-identified or re-identified).
