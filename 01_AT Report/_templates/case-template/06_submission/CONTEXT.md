# 06_submission — compile evidence and lodge

One job: package the report and its evidence for submission, and confirm it's ready.

## Inputs
- Working (this run): ../05_report-draft/output/report.md
- Reference (every run): ../../../references/report-block-structure.md (appendix structure)
- Reference (every run): ../../../references/submission-compliance-checklist.md

## Process
1. Build the appendix manifest: appendices hold original supporting evidence only (outcome-measure raw data, logbooks, feedback statements, medical letters, quote/product info) — never a restated summary of something already in the report body.
2. Run every item on `submission-compliance-checklist.md` before lodging.
3. Package as report + one combined supporting-evidence PDF (two files, clearly named), or as individually numbered PDFs if the recipient prefers — see the file-naming convention in `report-block-structure.md`.

## Outputs
- appendix-manifest.md → output/
- (submission itself is external — record the date and recipient here once sent)

## Human check
Final sign-off: is it clearly "reasonable and necessary," are cheaper alternatives ruled out on the record, is functional impact strong, is risk-of-non-funding stated. This gate exists specifically to prevent unpaid rework from a rejected submission.
